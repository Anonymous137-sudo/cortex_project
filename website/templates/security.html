Security

CryptEX web delivery is built to be domain-ready and locked down by default.

This site stack avoids the usual brochure-site shortcuts and treats the public web presence as part of the project's operational surface.

Application-level controls

  • Trusted host allowlist
  • Optional canonical domain redirect
  • Optional HTTPS enforcement
  • Strict CSP and defensive browser headers
  • Basic request rate limiting

Operational guidance

  • Reverse-proxy deployment through Caddy
  • Domain-specific environment configuration
  • Dedicated security.txt disclosure path
  • Explicit download attachment behavior
  • Canonical sitemap and robots metadata
